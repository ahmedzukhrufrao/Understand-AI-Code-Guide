# Contributing to Development Log Guide

Thank you for your interest in contributing! This guide helps non-coders understand AI-generated code, and your contributions make it better for everyone.

## How to Contribute

### Reporting Issues

Found a typo, unclear explanation, or have a suggestion? Open an issue!

**Good issue reports include:**
- Clear description of the problem or suggestion
- What you expected vs what you found
- Any relevant examples or screenshots

### Improving Documentation

- Fix typos or grammar
- Clarify confusing explanations
- Add more examples
- Improve templates
- Expand FAQ section

### Adding Examples

Share your own development log entries (anonymized if needed):
- Good examples of log entries
- Real-world use cases
- Different project types

### Suggesting Improvements

Have ideas for:
- Better rule file structure
- Additional templates
- More examples
- Clearer explanations

## Contribution Process

1. **Fork the repository**
2. **Create a branch** for your changes
   ```bash
   git checkout -b improve-examples
   ```
3. **Make your changes**
4. **Test your changes** (if applicable)
5. **Commit with clear messages**
   ```bash
   git commit -m "Add example for error handling"
   ```
6. **Push and create a Pull Request**

## Code of Conduct

- Be respectful and inclusive
- Help others learn
- Provide constructive feedback
- Keep explanations beginner-friendly
